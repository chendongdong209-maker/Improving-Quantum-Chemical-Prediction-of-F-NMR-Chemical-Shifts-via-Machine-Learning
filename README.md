# Improving Quantum-Chemical Prediction of 19F NMR Chemical Shifts via Machine Learning

This repository contains the curated data, notebook workflows, and publication-facing outputs for our study on combining DFT-computed isotropic shielding constants with machine learning to improve `19F` NMR chemical-shift prediction.

The cleaned project is organized around four notebooks:

1. `test.ipynb`
   Main tabular workflow on the full curated dataset: data cleaning, feature engineering, baseline benchmarking, hyperparameter refinement, and shielding-aware model comparison.
2. `test_class1.ipynb`
   Regression-mixture class partitioning, Class 1 subset construction, class-specific modeling, and cross-instrument calibration workflow.
3. `test_gcn.ipynb`
   Shielding-aware GCN workflow, including graph construction, Optuna search, final training, and GCN-based comparisons.
4. `draw.ipynb`
   Centralized figure-generation notebook for the main manuscript figures.

## Data Convention

- `data/raw`
  Original input datasets collected from the literature or external measurements.
- `data/processed`
  Cleaned datasets and engineered feature matrices used by the notebooks.

The core processed datasets used by the main workflows are:

- `data/processed/cleaned_shift_data.csv`
- `data/processed/cleaned_shift_data_with_shield.csv`
- `data/processed/cleaned_shift_data_with_shield_class1.csv`
- `data/processed/features_dataset.parquet`
- `data/processed/features_dataset_with_shield.parquet`
- `data/processed/features_dataset_with_shield_class1.parquet`
- `data/processed/features_dataset_60M_NMR.parquet`

## Results Convention

- `results/figures`
  Main-text figures and workflow figures.
- `results/tables`
  Benchmark tables, rankings, classification summaries, parity summaries, and exported manuscript tables.
- `results/models`
  Final ExtraTrees artifacts retained for the GitHub snapshot.
- `results/gcn`
  GCN checkpoints, loaders, encoders/scalers, and GCN-specific tables and figures.
- `results/supplementary`
  Supplementary-information figures and supporting tables.

Only the compact final ExtraTrees models are versioned in the GitHub-ready snapshot. Oversized intermediate tree ensembles were intentionally omitted because they can be regenerated by rerunning the notebooks but are awkward to host directly in a standard GitHub repository.

## Recommended Execution Order

1. Run `test.ipynb`.
2. Run `test_class1.ipynb`.
3. Run `test_gcn.ipynb`.
4. Run `draw.ipynb` to regenerate manuscript figures from the saved outputs.

All notebook paths are relative to the repository root, so the workflows can be executed after opening Jupyter in this directory.

## Environment

Install the core Python dependencies with:

```bash
pip install -r requirements.txt
```

For chemistry-focused environments, a conda-based setup is often more reliable for `rdkit`, `torch`, and `torch-geometric`. If your platform-specific PyG installation differs, follow the official installation instructions for PyTorch Geometric after installing the base scientific stack.
